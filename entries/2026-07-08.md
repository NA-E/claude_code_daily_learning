**CONCEPT**

**Harness Engineering, Defined: The Environment Layer Beneath the Prompt**
*Source: AI Agents Simplified (aiagentssimplified@substack.com, "If You're Confused About the 3 New Terms in AI Agents, Read This...")*

Harness engineering — coined by Terraform co-creator Mitchell Hashimoto in a February 2026 post — means building permanent fixes into an agent's environment (tools, validation, stop conditions) instead of correcting mistakes in chat each time. Cited example: an OpenAI team wrote zero code themselves for five months, letting Codex produce 1 million lines and 1,500 merged PRs, by solving four environment problems — a structured `docs/` directory for shared conventions, self-run tests with a concrete standard (services must start in under 800ms), linter-enforced dependency structure to stop architectural drift, and scheduled background agents that scan for tech-debt deviations and open their own refactor PRs.

*Why it matters for you*: A precise vocabulary and a concrete checklist (docs/ directory, self-verification standard, linter-enforced structure, scheduled drift-scanning) for hardening any Claude Code setup you build for a client, not just your own.

*Action*: Pick one of the four fixes — start with a `docs/` directory Claude Code must consult before making architectural decisions — and add it to your next client repo setup instead of re-explaining conventions in every session.

**NEWS**

**Google Patches "Rogue Agent" Flaw That Could Let One Compromised Dialogflow Bot Hijack Others**
*Source: The Neuron (theneuron@newsletter.theneurondaily.com, "Enterprise chatbots have a new security problem")*

Varonis Threat Labs disclosed a vulnerability (nicknamed "Rogue Agent") in Google Cloud's Dialogflow CX Code Blocks feature: agents in the same GCP project shared a Cloud Run execution environment, so one edit-permission compromise could let an attacker override a key execution file, read conversation history and session details, and force a chatbot to return attacker-chosen messages — including fake "reauthenticate" prompts to harvest credentials. Google issued an initial fix in April 2026 and fully resolved it in June; Varonis found no evidence of real-world exploitation.

*Why it matters for you*: A concrete illustration that agent security failures often live in shared infrastructure plumbing, not prompt injection — relevant context if you ever architect a multi-agent or multi-tenant Claude Code setup for a client where agents might share compute or credentials.

*Action*: If you build or advise on any multi-agent client system, ask explicitly whether each agent/bot runs in an isolated execution environment or shares one with others — this is the exact configuration Varonis found exploitable.
