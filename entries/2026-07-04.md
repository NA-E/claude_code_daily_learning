*Source: NLPiation #39 ("Memory Meets Autonomy", nlpiation@substack.com)*

**CONCEPT**

**StructMem — Event-Based Structured Memory Beats Flat and Graph Memory at 1.9M vs 35M Tokens**
*Source: NLPiation #39 (same issue)*

Binds each message into a timestamped "event," then consolidates related events into summaries once a buffer fills, retrieving both the summary and the raw events together. Hits ~76.8% overall accuracy (about 2% over baseline) with clear gains on temporal and multi-hop reasoning, using roughly **1.9M tokens versus over 35M tokens** for a system like Mem0g on the same task — far fewer API calls.

*Why it matters for you*: This is a concrete architecture behind July 2's McFarland rule ("give it a place to write things down") — StructMem's event-binding-plus-consolidation approach is a real design for that memory file, not just an unstructured markdown log.

*Action*: If any Claude Code skill or automation you run accumulates memory notes across many sessions, read StructMem's event-binding approach before your markdown lessons file grows unwieldy and hard to retrieve from.
