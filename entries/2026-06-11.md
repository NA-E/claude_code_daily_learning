### **CONCEPT**

**Fable 5 CLAUDE.md Audit Prompt — Your Knowledge Layer Was Written for a Weaker Model (Product Compass / Pawel Huryn)**
Huryn's most important finding: Fable 5 autonomously read his 320-line CLAUDE.md and 166-file / ~300K-word knowledge layer and flagged contradictions mid-task, without being asked. Five categories of issues it found: (1) **hardcoded wrong dates** (a rule file said "today is 2026-05-24" from May, never corrected); (2) **a rule documented with the pattern it bans** (em-dash ban written using em dashes); (3) **drifted calibration constants** (audience threshold said <50K followers, actual audience is now 75K); (4) **guardrails for failure modes the new model doesn't have** (rules like "never delegate judgment-heavy work to cheaper models" that exist because Opus needed them); (5) **the same rule in three files** (three drift surfaces). His conclusion: "a knowledge layer written by a weaker model has a flaw no amount of review can fix. Every line an agent added passed review by the model that added it, by definition." His audit prompt, verbatim:

```
Read your own instruction files (CLAUDE.md, skills, rules, memory files) end to end.

1. Where do they contradict each other? Quote both sides.
2. Which rules exist to manage a weaker model: guardrails for failure
   modes you don't have, recipes for things you no longer need spelled
   out, hardcoded facts that have drifted? List them with file:line.
3. Which rules teach by bad example: documents that violate the
   patterns they prescribe?
4. What would you delete? What would you keep exactly as is, and why?

Don't fix anything yet. Report first. I decide what gets cut.
```

*Source: Product Compass / Pawel Huryn (huryn@substack.com), June 11, 2026*
**Why it matters for you:** This repo has been building a CLAUDE.md-backed knowledge layer since 2026-05-19 (CLAUDE.md context foundation entry). Every entry since has added instructions, skills, and context rules written with Opus 4.8 or earlier models in mind. Fable 5 can now read and evaluate those instructions — and may be held back by guardrails that no longer apply to it.
**Action:** Open Claude Code with Fable 5. Paste the audit prompt above. Run it against your CLAUDE.md and any skills or context files. Do not delete anything yet — review the report first. The report itself is the deliverable today.

**Claude Subagents via YAML — Delegate Bulk Work to Haiku, Keep Fable 5 for Orchestration (Nate Herk / AI Automation Society)**
Nate Herk published a tutorial on sub-agent architecture for Claude: the main Claude session (Fable 5, Opus-class) delegates bulk parallelizable work to separate sub-sessions running cheaper models (Haiku-class, $0.25/M input). The mistake most people make: running everything through one session. Context fills up, you pay Fable 5 rates for tasks Haiku can handle, nothing runs in parallel. Sub-agents fix all three. The configuration mechanism is a YAML file that tells Claude when and how to invoke sub-agents. The practical pattern: orchestrator holds the plan and quality bar → sub-agents execute parallel chunks → orchestrator validates output. Real example from his community: Santella Austin built a working email-to-Google-Sheets automation in 24 hours from zero prior experience, self-debugged a row-insertion bug using Claude.
*Source: AI Automation Society / Nate Herk (nate@aiautomationsociety.ai), June 11, 2026*

The adversarial diagnostic pattern from 2026-06-08 spawned parallel agents — those parallel investigator agents should run on Haiku ($0.25/M), with only the adversarial validator running on Fable 5. That single architectural decision could reduce diagnostic workflow costs by ~80%.
