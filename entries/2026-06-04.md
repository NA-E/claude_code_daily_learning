### **CONCEPT**

**Temporal Knowledge Graphs — Why Vector RAG Fails as Agent Memory (The Neural Maze)**
Issue #04 of The Neural Maze (theneuralmaze@substack.com) explains the core problem with vector-based agent memory using a precise example: a user says "I moved to Madrid last month" — both the old and new cities score high on semantic retrieval, and the agent cannot determine which fact is current. "That's not a memory. That's a filing cabinet with no concept of time, no concept of who or what, and no concept that facts can die." Knowledge graphs solve this with 4 properties vector stores lack: (1) **Explainable reasoning** — the answer path is a literal graph path you can read aloud; (2) **Multi-hop reasoning** — chaining 3+ facts without relying on all chunks hitting top-k; (3) **Entity resolution** — two people named "Marta" can be reconciled or kept distinct; (4) **Flexible schema** — new entity/relationship types added without migration. The key leap is **bi-temporal edges**: every relationship carries both when it became true and when the system learned it, so superseded facts are marked as expired rather than deleted.
*Source: The Neural Maze Substack (theneuralmaze@substack.com), Issue #04, 2026-06-04*
**Why it matters for you:** This digest has no persistent memory between runs — each session re-reads prior entries to infer patterns. A Graphiti-backed memory layer would let the digest agent accumulate entity relationships over time (e.g., knowing you track Anthropic valuation across 6 entries) without re-reading the entire `entries/` folder every run.
**Action:** Read the full issue at theneuralmaze.substack.com. Sketch a data model: what entities and relationships would be worth tracking in a knowledge graph for your AI learning? (person → teaches → skill, tool → connects-to → MCP, model → benchmarks-at → score + date)

### **TOOL**

**Graphiti — Incremental Knowledge Graph Memory for Agents (Zep, github.com/getzep/graphiti)**
The runnable implementation from The Neural Maze: Graphiti (from Zep) ingests new information as discrete **episodes** and incrementally folds each into a Neo4j graph without full recomputation. Uses a **bi-temporal model** (event timestamp + ingestion timestamp). Hybrid retrieval: vector (semantic) + BM25 (keyword) + graph traversal, with sub-second P95 latency. Cost reality: every episode costs an LLM call (~0.5–2 seconds latency + token cost). Setup: Python 3.10+, Docker + Docker Compose (Neo4j runs in Docker, no manual install), OpenAI API key (swappable). Triple syntax: `(Luis, WORKS_WITH, Miguel)`. Each user gets their own `group_id` namespace — no cross-user memory bleed. You can watch nodes/edges appear in Neo4j Browser in real time.
*Source: The Neural Maze Substack (theneuralmaze@substack.com), Issue #04, 2026-06-04*
**Why it matters for you:** This is the first concrete, runnable implementation for persistent agent memory that appeared in this repo. The Dynamic Workflows feature (2026-05-30) spawns 1,000 parallel subagents — but they all forget each other when the session ends. Graphiti is what closes that gap.
**Action:** `pip install graphiti-core` and run the demo assistant from the newsletter. Set `group_id` to your name. Chat for 5 minutes, then open Neo4j Browser and observe the graph. One session is enough to evaluate whether this belongs in your stack.
