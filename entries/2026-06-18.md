### **WORKFLOW**

**Review AI-Generated Code by Risk, Not by Size**
Rahul Sengottuvelu (head of applied AI at Ramp) published a mental model this week endorsed by Boris Cherny (creator of Claude Code) and Elvis Saravia (AI educator). The core: stop treating all AI-written code the same. A 12-line change to the login system is more dangerous than a 1,200-line settings page change. Classification: (1) **High risk** — auth, identity, payments, data access, network access, PII, production DB writes → keep changes small enough for line-by-line review; (2) **Medium risk** — business logic, user-facing behavior, integrations; (3) **Low risk** — UI, formatting, internal tooling → verify empirically with tests, feature flags, or shadow mode. Boris Cherny added: the future workflow is Claude Code + advanced model + a verifier in a loop. Elvis Saravia warned: blind autonomous loops don't work without guardrails.
```
Review this AI-generated code by risk, not by size.
Classify the change:
1. High risk: auth, identity, payments, data access, PII, security, production DB writes.
2. Medium risk: business logic, user-facing behavior, integrations, performance.
3. Low risk: UI, formatting, internal tooling.
Then tell me: what could go wrong? what needs line-by-line review? what can be verified empirically? should this run behind a feature flag or shadow mode?
```
- **Source:** The Neuron (theneuron@newsletter.theneurondaily.com)
- **Why it matters for you:** This prompt makes AI code review systematic rather than ad-hoc. It's directly usable inside Claude Code as a pre-merge check. GSTACK's `/cso` and `/review` commands implement this logic — pairing this mental model with GSTACK makes review both structured and fast.
- **Action:** Add this classification prompt to your CLAUDE.md as a required pre-merge step for any code touching auth, payments, or data access.
