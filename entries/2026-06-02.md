**Codex Goals Framework — 6-Part Structure for Unattended Agent Runs (5h45m Tested)**
Claire Vo published a Codex Goals walkthrough (via Lenny's Newsletter) documenting a 5h45m unattended run: cleaned 3,900 emails down to 68, fixed hundreds of Sentry errors by having the agent categorize, repair, and replay historical examples. The framework applies to any agentic AI. Six parts every Goal must include: (1) **Outcome** — what should be true when done; (2) **Verification** — how to test it; (3) **Constraints** — what cannot regress; (4) **Boundaries** — which tools/files the agent may use; (5) **Iteration policy** — how to try again on failure; (6) **Stopping condition** — when to ask for help. Conversion prompt: `"Turn this task into a Goal an AI agent can run without babysitting. Task: [describe]. Write: 1. Outcome 2. Verification test 3. Constraints that cannot regress 4. Files/tools/systems the agent may use 5. Iteration policy 6. Stopping condition for asking me to step in."`
*Source: The Neuron Daily (theneuron@newsletter.theneurondaily.com), citing Lenny's Newsletter, June 2, 2026*
**Why it matters for you:** On May 16 you tracked `/goal` for unsupervised Claude Code loops. On May 30, Dynamic Workflows showed 1,000 parallel Claude Code subagents. Today's Goals Framework is the missing prompt standard that makes those loops actually work end-to-end. The Stopping Condition part (when to ask for help) is what prevents a 5-hour run from hitting a wall silently and wasting an entire session.
**Action:** Take your next Claude Code task and run it through the Goals conversion prompt before typing a single command. Write out item 2 (Verification) first — how will you know the task is actually done? Getting that one item right eliminates 80% of direction failures before they happen.
