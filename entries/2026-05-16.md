### **FEATURE**

**`/goal` — Claude Code Unsupervised Loop with Verifiable Success Condition**
Claude Code (and OpenAI Codex) both shipped `/goal` this week. You give the agent one concrete, verifiable success condition — a passing test, a live URL, a file diff — and it loops: run → check → run → check, until the condition is met. No mid-session babysitting. Recommended structure per RoboNuggets newsletter: (1) **Definition of Done** — one concrete sentence, no fuzzy words like "clean" or "good"; (2) **Out of Scope** — at least two things you explicitly don't want built; (3) **A Check Each Turn** — a command, screenshot, or small test the agent can run itself; (4) **Turn Limit** — 25 for small jobs, 50 for medium, 100 for large. Source: self-sent Claude & Anthropic Daily Digest, May 15 (citing RoboNuggets newsletter).
**Why it matters for you:** This is the runtime companion to Claude Code Routines (May 12 entry). Routines schedule execution; /goal governs autonomous behavior within a session. Together they close the loop: cloud-triggered → self-verifying → loop until done.
**Action:** Today, pick one well-defined task (e.g., "all tests pass in this repo" or "landing page HTML validates with zero errors"). Open Claude Code, type `/goal`, then paste your four-element structure. Set turn limit to 25. Watch the first loop iteration and refine your success condition if it proves ambiguous.

**Agent Architecture Files — AGENTS.md + TASK.md + MEMORY.md Structure**
The Creators' AI Weekly guide "How AI Agents Are Built in May 2026" detailed the three-file structure now standard in production agent projects: **AGENTS.md** (role definitions — what each agent can and cannot do, which tools it has access to, how it hands off to the next agent); **TASK.md** (the current job — one active task per file, success condition, turn limit, out-of-scope list); **MEMORY.md** (persistent state — decisions made, facts confirmed, patterns observed across sessions). The guide emphasizes that the most common failure mode is context loss between runs, and that MEMORY.md populated correctly prevents 80% of repeated-mistake patterns. Source: Creators' AI Weekly (May 15).
**Why it matters for you:** This is the structural companion to /goal. You define the TASK.md success condition, /goal runs until it's met, and MEMORY.md captures what was learned. This directly extends the CLAUDE.md version control workflow from May 12.
**Action:** Create a `agent-structure/` folder in this repo. Add `AGENTS.md` with one agent role (the digest agent: what it does, what tools it uses, what it must not do). Add `TASK.md` with today's success condition already filled in. Add `MEMORY.md` with three observations from this digest run. This is a 15-minute setup that pays off in every future session.
