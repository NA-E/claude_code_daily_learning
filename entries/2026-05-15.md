**/goal for Claude Code + Codex — Verifiable Stopping Condition, Install from GitHub**
RoboNuggets (Jay, robonuggets.blog, May 15) surfaced `/goal` — a recently shipped Claude Code and Codex feature that lets an agent run unsupervised for hours with a verifiable done condition. The mechanism: you give the agent a concrete success condition, then it loops (run → check → run → check) until the condition is met. The 5-element plan structure for effective `/goal` use: (1) **Definition of Done** — one short sentence with no fuzzy words ("the page saves a CSV with the current filters"); (2) **Out of Scope** — at least 2 explicit exclusions to prevent wandering; (3) **A Check Each Turn** — a command, screenshot, or small test the agent can run itself; (4) **Turn Limit** — 25 turns for small jobs, 50 for medium, 100 for large (do not exceed); (5) iteration between checks. Install the plan-for-goal skill: `git clone https://github.com/robonuggets/plan-for-goal`. Source: RoboNuggets (robonuggets.blog, May 15).
**Why it matters for you:** This digest already runs as an autonomous agent. `/goal` would let it define its stopping condition precisely — "digest file written, INDEX.md updated, commit pushed, PR created" — rather than relying on implicit task completion. The turn limit prevents runaway loops.
**Action:** Run `git clone https://github.com/robonuggets/plan-for-goal` and follow the README. Then write a `/goal` spec for this digest: DoD = "entries/YYYY-MM-DD.md committed and pushed"; Out of scope = "do not send emails, do not modify entries other than today's"; Check each turn = "verify file exists at entries/YYYY-MM-DD.md"; Turn limit = 50. Add this spec to your CLAUDE.md for the digest Routine.

### **CONCEPT**

**Goal + Hook Architecture — Agent Continuity Pattern for Normal Work**
The Neuron Daily's AI Skill of the Day (May 15) formalized the Goal + Hook architecture for non-developer use: a **goal** is a verifiable finish line ("what counts as done, in concrete terms"); a **hook** is a just-in-time rule that runs automatically at a specific moment. Developer examples (from Codex): scan for secrets before commit, run validators, create memories. Normal-work translations: "before sending an email, check whether I sound too harsh" / "before booking travel, check my budget" / "before filing an expense, attach the right receipt and categorize it correctly." The prompt template for converting any task into a goal+hook agent workflow: `Help me turn this task into a long-running agent workflow: [TASK]. Define: (1) the goal — what counts as done; (2) checkpoints — where the agent should stop and ask me; (3) hooks — rules that run automatically; (4) risky steps — anything public, expensive, irreversible; (5) progress updates — what I should see while work is running.` Source: The Neuron Daily (theneurondaily.com, May 15).
**Why it matters for you:** This directly extends May 12's Evals framework — where Evals define when a prompt is *done*, Goal+Hook architecture defines when an *agent session* is done. The two concepts are complementary stops for the same problem: "how do I know AI finished the job correctly?"
**Action:** Use the template above on your next agent task before starting it. Write the goal first (one sentence, no fuzzy words), list two out-of-scope items, name one hook that should auto-run. This 5-minute pre-task spec replaces ad-hoc steering mid-session.
