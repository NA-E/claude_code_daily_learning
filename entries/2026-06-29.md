**GLM-5.2 in Claude Code — Exact Undocumented Setup**

Claire documented the exact configuration that isn't in any official docs. For Cursor: override the OpenAI base URL to `openrouter.ai/api/v1/cursor` (the `/cursor` suffix is not documented anywhere — you'll get errors without it), then add `z-ai/glm-5.2` as a custom model with your OpenRouter API key. For Claude Code: make 2 environment variable changes (`OPENAI_API_KEY` → your OpenRouter key; `OPENAI_BASE_URL` → `https://openrouter.ai/api/v1`) and edit `~/.claude/settings.json` to add the model. Total setup time: under 1 hour once you have the exact strings.
- **Source:** Lenny's How I AI (Substack), June 29 — Claire Vo's GLM-5.2 episode
- **Why it matters:** The barrier was never the model itself — it was that the routing config wasn't documented. Now it is.
- **Action:** `export OPENAI_API_KEY=<openrouter_key> && export OPENAI_BASE_URL=https://openrouter.ai/api/v1` in your shell profile, then add `"model": "z-ai/glm-5.2"` to `~/.claude/settings.json`. Test with a long agentic task.
