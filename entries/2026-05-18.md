### **WORKFLOW**

**HTML is the New Markdown — Thariq Shihipar's Claude Code System (Anthropic)**
Thariq Shihipar, engineer on Anthropic's Claude Code team (recorded live at the "Code with Claude" event), has replaced Markdown with HTML for all plans, status updates, and agent communications. His reasoning: as Claude's context windows expanded and plans grew to thousands of lines, "HTML makes it actually possible to engage with the content rather than just skimming or ignoring it entirely." He stopped reading long plans and "started asking Claude to edit them instead." His prompt is deliberately simple: *"Create an HTML file with a plan. Help me visualize. Include excerpts, mockups, code, whatever is needed to give me maximum context."* The phrase "whatever is needed" signals trust and gives Claude permission to make decisions without over-constraining it. He sends his manager "weekly HTML status updates because they're more likely to actually get read than Markdown or plain text." Full blog post: https://www.chatprd.ai/how-i-ai/claude-code-anthropic-thariq-shihipar-on-replacing-markdown-with-html
Source: How I AI (lenny+how-i-ai@substack.com, May 18).
**Why it matters for you:** Every plan, TASK.md, and AGENTS.md file you've built since May 10 is in Markdown. The person who built Claude Code is telling you Markdown plans are a productivity floor, not a ceiling.
**Action:** Open Claude Code. Paste your most recent complex task as a Markdown plan. Then type: *"Convert this into an interactive HTML file. Include mockups, color-coded sections, and any visual aids that would help me review and edit it."* Save the output as `plan.html` and open it in a browser. That's the experiment. Do it once before committing to the format change.
