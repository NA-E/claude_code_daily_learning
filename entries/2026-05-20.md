### **CONCEPT**

**Transition Failure Matrix — Agent Debugging Visualization for Multi-Step Failures (Hamel Husain / Parlance Labs)**
When an agent has many steps, tool calls, and branching paths, reviewing individual traces doesn't reveal systemic patterns. The Transition Failure Matrix fixes this: a grid where rows = "From State" (last successful step) and columns = "To State" (where the first failure occurred). Each cell = failure count. Example from the email: in a text-to-SQL agent, GenSQL → ExecSQL causes **12 failures**; DecideTool → PlanCal causes only **2**. Now you know where to focus debugging. Build steps: (1) define states — list every tool call or reasoning step; (2) create the matrix; (3) populate cells — start with failure counts, extend to failure rates, latency percentiles, or token costs as needed. Critical distinction: a state with many failures isn't the root cause — ExecSQL fails after LLM-generated SQL but succeeds on user-specified SQL. The "From State" column separates these cases. Flashcard image: https://hamel.dev/notes/llm/evals/flashcards/11-transition-matrix.png. Evals course with $1,250 off: https://maven.com/parlance-labs/evals?promoCode=newsletter-flashcards-25
Source: Hamel Husain / Parlance Labs (hamel_husain@parlance-labs.com), May 19.
**Why it matters for you:** May 12 established that evals give prompt engineering a stopping condition. Today's Transition Failure Matrix gives you the next layer: once your evals are running and failures appear, this is how you visualize *where* in the pipeline failures cluster. For any multi-step Claude Code routine (Routines from May 12, /goal loops from May 16), a transition matrix turns "it sometimes fails" into "it fails specifically at the GenSQL → ExecSQL transition 12 times." That's a fixable prompt, not a mysterious bug.
**Action:** For your most-used Claude Code routine, list all states (steps) on paper. Next time a failure occurs, log: which step was last successful, which step failed. After 5 failures, you'll have the seed of a transition matrix. The pattern will be visible before you hit 10 entries.
