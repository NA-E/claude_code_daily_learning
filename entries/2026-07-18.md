**FEATURE**

**Claude Skills, Fully Explained: It's One File Called SKILL.md, and You Install One by Typing a Sentence**

*Source: The AI Break (theaibreak@substack.com), July 17*

A paywalled tutorial's free portion laid out the actual mechanics behind Claude Skills, more precisely than this repo has captured before: a skill is a folder containing one file, `SKILL.md`, holding a name, a description, and instructions. Claude reads every skill's name and description continuously (near-zero cost); when a request matches a description, the full instructions load. To install: paste a skill block into Claude or Claude Code and say **"Create this as a skill."** — Claude writes and names the file itself. In Claude (web/app) it appears under Customize → Skills; in Claude Code it's live in the current session immediately. Adding "for all my projects" to the install request routes it to `~/.claude/skills/` so every project gets it; omitting that phrase scopes it to the current project only. Manual install: folder + `SKILL.md`, zipped, then Customize → Skills → + → Create skill → Upload a skill.

*Why it matters*: this repo has covered multiple pro- and anti-Claude-Skills takes (Every.to's skepticism July 16, this week's paywalled hype pieces) without ever nailing down the actual install mechanics in one place — this is the missing "how," independent of whether skills are worth building.

*Who it's for*: anyone building or maintaining a personal or team Claude Skills library.

*Action*: pick one recurring task you do manually, write it as a plain-English instruction block, paste it into Claude Code, and say "Create this as a skill, for all my projects" — that's the entire install process, and it's the cheapest possible test of Mike Taylor's July 16 question ("do I have proof this outperforms the model with no skill at all?").
