### **CONCEPT**

**Skills Supply-Chain Backdoor Attack: 37K-Star GitHub Collection, 26K Agents Reached, All Scanners Pass**
Cybersecurity company AIR built and released a Claude skill containing a hidden backdoor to demonstrate supply-chain risk in the emerging skills ecosystem. The skill was accepted into a popular GitHub collection with 37,000 stars, promoted on Instagram, and reached 26,000 agents. AIR then changed the external instructions the skill pointed to — triggering code download and execution on all 26,000 affected agents. Every security scanner still marked the skill safe because the skill file itself never changed; only the external instructions did. The attack vector: skills that call external URLs for their instructions can be hijacked by changing the URL's response, not the skill file.
- **Source:** Every.to (hello@every.to) — "Can AI Learn Good Judgment?", June 24, 2026 (citing Niv Hoffman, "The Story of Skills")
- **Why it matters for Nourin:** The June 8 entry documented adversarial agent diagnostics and multi-agent trust issues. This is the first confirmed skills supply-chain attack pattern in the wild. If you install any Claude skill from a GitHub collection or third-party source, you are trusting not just the skill file but every external URL that skill loads at runtime. The attack is invisible to static scanners.
- **Action:** Audit every Claude skill you've installed: open each skill file and check whether it calls any external URL for instructions or context. Skills that load from external URLs (vs. self-contained files) carry backdoor risk. Prefer self-contained skills or skills from sources where you can pin a specific commit hash and verify external URL targets.
