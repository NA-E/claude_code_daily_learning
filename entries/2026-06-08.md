## Top Thing to Implement Right Now

**ONE ACTION → HIGH LEVERAGE → DO TODAY**

**Build the multi-agent adversarial diagnostic pattern in Claude Code — one workflow file, applicable to any domain.**

Here is the exact architecture to implement:

```
# adversarial-diagnostic.md (CLAUDE.md project skill)

You are running an adversarial diagnostic. The goal: identify the single most
likely root cause of [PROBLEM] from a set of competing hypotheses.

## Step 1 — Parallel Investigation (run these as separate subagents, each blind to the others)

Agent A — Hypothesis: [CAUSE 1]
You have access ONLY to: [DATA SLICE A]
Reason from your data. What evidence supports or contradicts this hypothesis?
Do not reference any data outside your slice.

Agent B — Hypothesis: [CAUSE 2]
You have access ONLY to: [DATA SLICE B]
Reason from your data. What evidence supports or contradicts this hypothesis?
Do not reference any data outside your slice.

Agent C — Hypothesis: [CAUSE 3]
You have access ONLY to: [DATA SLICE C]
Reason from your data. ...

## Step 2 — Adversarial Validation (single agent, full dataset)

You have the full dataset and all three theories from the agents above.
For each theory: find the single strongest piece of evidence that DISPROVES it.
A theory survives only if the full dataset cannot kill it.
Rank surviving theories by confidence. State the winning diagnosis in one sentence.
```

**Why this over everything else today:**
- 2026-06-06's top action (Grill Me) solved context quality — Claude knowing you. Today's pattern solves *reasoning quality* — Claude thinking through problems without confirmation bias.
- The parallel blind-agent architecture eliminates the #1 failure mode in AI diagnosis: the first plausible explanation anchors all subsequent reasoning. Each agent reasons independently, so the adversarial validator is comparing genuinely separate analyses.
- This applies to ANY multi-hypothesis situation you face this week: a client's underperforming funnel, a bug with multiple possible causes, a content strategy with competing angles, a pricing decision with competing risk factors.
- Mike's concrete test result: gut said "creative fatigue" (would have killed the best-performing ad). The workflow said "budget delivery broadening" (required a spend rollback instead). That's a $X in wasted ad spend prevented by the correct diagnosis.
- **Set up time: 20 minutes.** Copy the architecture above into a new CLAUDE.md project. Fill in [PROBLEM], [CAUSE 1-3], and [DATA SLICE A-C] from one real problem you're currently trying to diagnose. Run it.
