### **WORKFLOW**

**AI Secretary Workflow (Claude Code) — Slack + Gmail + Calendar → MonoNote.md Daily Prioritized Task List**
The Rundown AI published a step-by-step guide: use Claude Code to build an AI taskmaster that checks Slack, Gmail, and your calendar every morning. Outputs a prioritized to-do list into MonoNote.md with date, source links, and status checkboxes. The automation learns your priority rules over time via task-rules.md. Steps: (1) Open Claude Code in a folder, ask it to create a skill that checks Slack, Gmail, and Calendar daily, prioritizes tasks high to low, and puts them at the top of MonoNote.md; (2) Tell the agent to create MonoNote.md and task-rules.md with your priority rules; (3) Run and review; (4) Ask the agent to automate it — it reviews yesterday's feedback, rolls over bumped tasks, updates the rules, generates the new list. Pro tip: create a weekly audit skill that finds repeated tasks and suggests automations. Source: The Rundown AI (news@daily.therundown.ai, May 25).
**Why it matters for you:** This is the "AI secretary" pattern made concrete and replicable. The MonoNote.md approach is particularly smart: it creates a persistent record of your daily task state that the agent can reference and learn from over time. Each day's list is a training signal for the next day's prioritization.
**Action:** Open Claude Code, create a new folder called `ai-secretary`, and paste this prompt: "Create a skill that reads my Gmail and Google Calendar each morning and outputs a prioritized to-do list to MonoNote.md with today's date, source links, and checkboxes. Also create task-rules.md where I can write my priority rules." Run it. The first version will be rough — spend 5 minutes editing task-rules.md, and tomorrow's list will be better.

**Felix Rieseberg's Sonnet vs. Opus Heuristic + "Go One Abstraction Layer Up" Pattern**
Anthropic's Felix Rieseberg (Claude Cowork engineering lead) gave two pieces of concrete guidance in the How I AI podcast. First, model selection: use **Sonnet** when the problem is well-scoped and the output is predictable. Reach for **Opus** when you need Claude to interpret what you actually want, not just what you said — when the task requires problem decomposition before execution. "Make me a floor plan with units" → Sonnet. "Help me figure out how to organize my life" → Opus. Second, the abstraction pattern: every time you catch yourself doing tedious work, stop and ask how Claude could do it instead — then ask how Claude could *figure out what to do* without your telling it. He demonstrated: manually entering furniture → Claude figures out what furniture he has → Claude finds it in his emails. Each step up is genuine leverage. Source: How I AI, Lenny's Newsletter (lenny+how-i-ai@substack.com, May 25).
**Why it matters for you:** The model heuristic reduces cost (Sonnet is meaningfully cheaper) while preserving Opus for the tasks that actually need interpretive intelligence. The abstraction pattern is the underlying cognitive move that makes every tool 10x more useful without any new tooling.
**Action:** For your next 10 Claude interactions, classify each one before sending: Is the output predictable given my prompt? If yes → Sonnet. If you're not sure what you actually need → Opus. After 10 interactions, you'll know your actual task distribution and can tune your defaults.
