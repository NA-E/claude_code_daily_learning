**Add the four Compound Engineering slash commands to your CLAUDE.md as skill stubs.**

Here is the exact text to add to your CLAUDE.md under a `## Skills` or `## Slash Commands` section:

```markdown
## Slash Commands

### /ce-brainstorm
Think through the feature or task. Produce a requirements doc with:
- Goal (one sentence)
- Constraints and edge cases
- Open questions that need resolving before implementation
Do NOT write code. Output the requirements doc only.

### /ce-plan
Research the codebase for relevant patterns, existing utilities, and potential conflicts.
Produce a step-by-step implementation plan with:
- Files to create or modify
- Key functions or classes to add
- Test cases to cover
Do NOT write code. Output the plan only.

### /ce-work
Execute the plan from /ce-plan. Follow existing project patterns.
Write production-ready code. After writing, run any available linters or tests.

### /lfg
Run the full loop: /ce-brainstorm → /ce-plan → /ce-work → open a PR description.
Use for well-scoped features where all context is already available.
```

- The commands enforce the "plan before code" discipline that prevents the common failure mode where Claude rewrites things it shouldn't touch.
- This takes 10 minutes to add to CLAUDE.md. The leverage compounds every time you run a Claude Code session from now on.
