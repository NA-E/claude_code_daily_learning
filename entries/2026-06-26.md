**CONCEPT**

**Loop Engineering — The 3-Loop Framework from Claude Code's Creator**
Boris Cherney (Claude Code's creator) and Peter Steinberger (OpenClaw's creator) both named "loop engineering" on social media, prompting Andrew Ng to write the lead letter in The Batch (June 26). Three nested loops: **(1) Agentic Coding Loop** — agent writes code, tests, iterates every few minutes until bug-free; no human needed; Ng built a complete typing-practice app in one weekend with an hour of uninterrupted agent runtime using a web browser autonomously. **(2) Developer Feedback Loop** — you examine the product and steer the agent over tens of minutes to hours; your role shifts from QA-finder to product decision-maker. **(3) External Feedback Loop** — friends, alpha testers, A/B tests over hours to weeks; feeds back into product spec. Ng: "With coding agents speeding up software development, more engineers are starting to play a partial product management role." Source: DeepLearning.AI The Batch, June 26 (Andrew Ng lead letter).
**Action:** Define your next project's spec + 3–5 acceptance criteria (evals) before opening Claude Code, then let Loop 1 run uninterrupted for 1 hour before you touch it.

## Top Thing to Implement Right Now

**ONE ACTION → HIGH LEVERAGE → DO TODAY**

**Run the 3-Loop Engineering workflow on a real project this weekend.**

Andrew Ng (citing Boris Cherney's concept) published exactly how to do it. Before you open Claude Code, write a one-paragraph spec + 3 acceptance criteria. Then:

```bash
claude --headless "Build [your spec]. You are done when: [criterion 1], [criterion 2], [criterion 3]. Test in a browser. Iterate until all criteria pass without my input."
```

Leave it running for **1 uninterrupted hour** (Loop 1 — don't touch it). Come back and use the product as a real user, then steer with one message per issue (Loop 2 — you're a PM now, not a debugger). Send to one friend for feedback (Loop 3). This is the methodology Claude Code's creator named today, and you already have everything required to run it — Claude Code, a browser, and a cron job. The spec + evals are the only thing you haven't been writing upfront. Start there.
